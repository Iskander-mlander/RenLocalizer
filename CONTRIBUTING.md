# 🚀 Contributing to RenLocalizer

First of all, **thank you** for being part of this mission. RenLocalizer is a community-driven project built on the principle of volunteerism and shared benefit.

---

## 🌟 1. The Spirit of Contribution
When you contribute here, you aren't just writing code—you are helping thousands of players and creators around the world connect through stories. We value your time, as this project exists because of people like you who give back.

## 📜 2. The Non-Commercial Covenant
> [!IMPORTANT]
> By contributing, you agree to these core principles:
> *   **Always Free:** RenLocalizer must remain 100% free. Your work will never be gated by a paywall.
> *   **Shared Legacy:** If you fork or maintain this project in the future, you must keep it open and free.
> *   **Altruism:** Donations for your time are okay, but selling the software is not.

---

## 🛠️ 3. Development Setup

### 📦 Quick Start
1. **Clone & Enter:**
   ```bash
   git clone https://github.com/Lord0fTurk/RenLocalizer.git
   cd RenLocalizer
   ```
2. **Environment:**
   ```bash
   python -m venv .venv
   ```
3. **Activate & Install:**
   ```bash
   # Windows: .venv\Scripts\activate | Linux: source .venv/bin/activate
   pip install -r requirements.txt
   ```

---

## 🎯 4. Contribution Guidelines

### ⚡ Quality over Quantity
We prefer **"Surgical Edits"**—small, precise changes that fix bugs or add value without disrupting the core architecture.

### 🏛️ Core Pillars
*   **Ren'Py Integrity:** Never break game syntax for translation convenience.
*   **User Experience:** Keep the UI clean (PyQt6/QML) and the CLI reliable.
*   **Syntax Guard:** Placeholders and technical tags are sacred; never corrupt them.

---

## 🧪 5. Testing & Validation
Before submitting your Pull Request, ensure the engine stays solid:
```bash
python -m pytest tests
```

---

## 🤝 6. Pull Request Process
1.  **Start a Dialogue:** For large features, open an **Issue** first to discuss the vision.
2.  **Stay Focused:** One PR = One feature or fix.
3.  **Document:** Update relevant docs if you change behavior.
4.  **Changelog:** Add your mark to `CHANGELOG.md`.

---
